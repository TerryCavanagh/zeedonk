package terrylib.bitmapFont

class BitmapFont

Holds information and bitmap glyphs for a bitmap font.

Constructor

new (name:String, bitmap:BitmapData)

Creates and stores new bitmap font using specified source image.

Fields

bitmap:BitmapData

Source image for this font.

bold:Bool

fontName:String

glyphs:Map<Int, BitmapGlyphFrame>

italic:Bool

read only lineHeight:Int

minOffsetX:Int

Minimum x offset in this font. This is a helper varible for rendering purposes.

read only size:Int

The size of the font. Can be useful for AngelCode fonts.

spaceWidth:Int

The width of space character.

Methods

dispose ():Void

Destroys this font object. WARNING: it disposes source image also.

prepareGlyphs (scale:Float, color:UInt, ?useColor:Bool, ?smoothing:Bool):BitmapGlyphCollection

Generates special collection of BitmapGlyph objects, which are used in RENDER_BLIT mode. These BitmapGlyph objects contain prepared (scales and color transformed) glyph images, which saves some CPU cycles for you.

Parameters:

scale

How much scale apply to glyphs.

color

color in AARRGGBB format for glyph preparations.

useColor

Whether to use color transformation for glyphs.

Returns:

Generated collection of BitmapGlyph objects. They are used for rendering text and borders in RENDER_BLIT mode.

Loads font data in AngelCode's format.

Parameters:

Returns:

staticfromMonospace (key:String, source:BitmapData, ?letters:String, charSize:Point, ?region:Rectangle, ?spacing:Point):BitmapFont

Loads monospace bitmap font.

Parameters:

key

Name for this font.

source

Source image for this font.

letters

The characters used in the font set, in display order. You can use the TEXT_SET consts for common font set arrangements.

charSize

The size of each character in the font set.

region

The region of image to use for the font. Default is null which means that the whole image will be used.

spacing

Spaces between characters in the font set. Default is null which means no spaces.

Returns:

Generated bitmap font object.

staticfromXNA (key:String, source:BitmapData, ?letters:String, ?glyphBGColor:Int):BitmapFont

Load bitmap font in XNA/Pixelizer format. I took this method from HaxePunk engine.

Parameters:

key

Name for this font.

source

Source image for this font.

letters

String of glyphs contained in the source image, in order (ex. " abcdefghijklmnopqrstuvwxyz"). Defaults to DEFAULT_GLYPHS.

glyphBGColor

An additional background color to remove. Defaults to 0xFF202020, often used for glyphs background.

Returns:

Generated bitmap font object.
